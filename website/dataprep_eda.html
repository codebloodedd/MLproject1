API Requirement (endpoint + example GET)

Data was collected from TMDB using two core endpoints:

  • GET https://api.themoviedb.org/3/movie/popular (collects pages of popular movies)
  • GET https://api.themoviedb.org/3/movie/{movie_id} (enriches each movie with runtime, budget, revenue, genres)

Example GET request (template):

https://api.themoviedb.org/3/movie/popular?api_key=YOUR_KEY&language=en-US&page=1

The API key is not shown publicly. The raw JSON response was converted to CSV and stored as raw data.

Cleaning Summary

  • Converted numeric fields (popularity, vote_average, vote_count, runtime, budget, revenue) to numeric types.
  • Filled missing numeric values with the median to remove NaNs while keeping all rows.
  • Standardized text fields (title/genres/language) and filled missing categories.
  • Capped extreme budget and revenue outliers at the 99th percentile for stability in plots and later models.
  • Created labels:
    • label_popular_top25: 1 if popularity in top 25% of the dataset.
    • label_high_rating: 1 if vote_average ≥ 7.0.

This cleaned dataset is used throughout the rest of the project. Standardized numeric columns support PCA and clustering, while the labels support supervised learning models. The early visualizations also guide later choices (e.g., handling skew and outliers, selecting feature transformations, and deciding which variables may be most informative).
